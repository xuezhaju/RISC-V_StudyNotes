## `lw` 和 `sw` 的语法（RISC-V）

这两条指令是RISC-V中**最核心的访存指令**，用于在**寄存器和内存之间传输数据**。

---

## 一、基本语法

```assembly
lw rd, offset(rs1)   # Load Word：从内存加载32位数据到寄存器
sw rs2, offset(rs1)  # Store Word：将寄存器中的32位数据存入内存
```

| 符号 | 含义 | 说明 |
| :--- | :--- | :--- |
| `rd` | 目标寄存器 | `lw` 把数据加载到这里 |
| `rs2` | 源寄存器 | `sw` 把这里的数据存出去 |
| `rs1` | 基址寄存器 | 存放内存地址的基址 |
| `offset` | 偏移量 | 12位有符号整数（-2048 ~ 2047） |

**计算公式**：`有效地址 = rs1 + offset`

---

## 二、具体例子

### `lw` 示例：从内存加载

```assembly
# 假设 x15 里存放着数组 A 的起始地址（比如 0x1000）
lw x10, 12(x15)   # x10 = 内存[x15 + 12]

# 如果 x15 = 0x1000，则从地址 0x100C 读4字节到 x10
```

### `sw` 示例：存入内存

```assembly
# 假设 x15 = 0x1000，x10 = 0x12345678
sw x10, 12(x15)   # 把 x10 的值存入地址 x15 + 12 = 0x100C

# 内存地址 0x100C 处现在变成 0x12345678
```

### 完整示例：数组访问

```assembly
# int A[100]; A[3] = 42; int b = A[3];
.section .data
A: .space 400          # 100个int，每个4字节

.section .text
la x15, A              # x15 = A的地址

# A[3] = 42
li x10, 42             # x10 = 42
sw x10, 12(x15)        # 存入 A[3]（偏移12 = 3*4）

# b = A[3]
lw x11, 12(x15)        # x11 = 从A[3]读取的值（现在=42）
```

---

## 三、偏移量的计算

在RISC-V中，`lw`/`sw` 的偏移量单位是**字节**。

| 数组索引 | 字节偏移量 | 计算公式 |
| :--- | :--- | :--- |
| A[0] | 0 | 0 |
| A[1] | 4 | 1 × 4 |
| A[2] | 8 | 2 × 4 |
| A[3] | 12 | 3 × 4 |

```assembly
# 访问结构体成员
struct Point {
    int x;   // 偏移 0
    int y;   // 偏移 4
};

# 假设 x15 = &p
lw x10, 0(x15)   # x10 = p.x
lw x11, 4(x15)   # x11 = p.y
sw x10, 8(x15)   # p.z = x10（假设z在偏移8）
```

---

## 四、对齐要求

RISC-V要求 `lw`/`sw` 的地址**必须是4的倍数**（按字对齐）。

| 地址 | 是否合法 | 说明 |
| :--- | :--- | :--- |
| 0x1000 | ✅ 合法 | 0x1000 ÷ 4 = 整数 |
| 0x1002 | ❌ 不合法 | 除以4余2，会触发地址不对齐异常 |
| 0x1004 | ✅ 合法 | 4的倍数 |

```assembly
# 错误示例（会崩溃）
la x15, unaligned_addr   # 假设这个地址不是4的倍数
lw x10, 0(x15)           # 硬件异常！
```

**为什么要对齐？** 数据通路设计时，内存接口通常是4字节宽，非对齐访问需要多次访存，RISC-V选择简化硬件，要求程序员/编译器保证对齐。

---

## 五、`lw` 与 `sw` 的指令编码

这两条指令的编码格式不同（这是第3章的内容）：

| 指令类型 | 指令 | 编码格式 |
| :--- | :--- | :--- |
| **I型** | `lw rd, offset(rs1)` | `[offset:11:0] [rs1:4:0] [funct3=010] [rd:4:0] [opcode=0000011]` |
| **S型** | `sw rs2, offset(rs1)` | `[offset:11:5] [rs2:4:0] [rs1:4:0] [funct3=010] [offset:4:0] [opcode=0100011]` |

你可以用 `riscv64-elf-objdump -d` 查看真实指令的编码：

```bash
# 写一段简单的代码
cat > test.s << 'EOF'
.section .text
.globl _start
_start:
    lw x10, 0(x15)
    sw x10, 4(x15)
EOF

riscv64-elf-gcc -c test.s -o test.o
riscv64-elf-objdump -d test.o
```

输出示例：
```
00000000 <_start>:
   0:   0007a503        lw      a0,0(a5)
   4:   00a7a223        sw      a0,4(a5)
```

左边 `0007a503` 和 `00a7a223` 就是机器码。

## 六、常见错误与调试

| 错误 | 原因 | 解决方案 |
| :--- | :--- | :--- |
| `lw` 地址不对齐 | 偏移量不是4的倍数 | 检查偏移量 |
| `sw` 没生效 | 目标地址在只读段 | 检查 `.section .data` 或 `.rodata` |
| 加载到错误的值 | 字节序误解 | RISC-V默认小端，低地址存低位 |

## 七、与你正在学的其他指令对比

| 指令 | 操作 | 数据传输方向 | 典型用法 |
| :--- | :--- | :--- | :--- |
| `addi x10, x11, 5` | 运算 | 寄存器→寄存器 | 计算中间值 |
| `lw x10, 0(x11)` | 加载 | **内存→寄存器** | 读变量/数组元素 |
| `sw x10, 0(x11)` | 存储 | **寄存器→内存** | 写变量/数组元素 |

**内核开发中的典型代码**（任务切换时保存寄存器）：

```assembly
# 保存当前任务的寄存器到内存（sw）
addi sp, sp, -256      # 栈上分配256字节
sw x1, 0(sp)           # 保存ra
sw x2, 4(sp)           # 保存sp
# ... 保存x3~x31 ...

# 恢复另一个任务的寄存器（lw）
lw x1, 0(sp)           # 恢复ra
lw x2, 4(sp)           # 恢复sp
# ... 恢复x3~x31 ...
```

如果你理解了 `lw` 和 `sw`，就掌握了内核中任务切换、内存管理的最底层操作。
