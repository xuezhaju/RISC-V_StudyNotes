## 汇编中的“标记”（Label）完全讲解

**标记是你在汇编代码中给某个内存地址起的“别名”或“标签”。** 它本身不是指令，不会变成机器码，只是告诉汇编器：“这个地方有个名字，后面的跳转指令可以用这个名字来指代这个地址。”

---

## 一、标记长什么样？

```assembly
loop:                    # 👈 这就是一个标记，名字叫 "loop"
    addi x9, x9, -1      # 这是实际指令
    bnez x9, loop        # 👈 跳转到标记 "loop" 所在的位置
```

**语法规则**：
- 标记名字后面要跟一个**冒号** `:`
- 可以单独占一行，也可以放在指令前面
- 名字只能由字母、数字、下划线组成，但**不能以数字开头**

**合法标记**：`start:`、`_loop:`、`label_1:`、`again:`
**非法标记**：`1_label:`（数字开头）、`my label:`（有空格）

---

## 二、标记的本质是什么？

| 层面 | 标记是什么 |
| :--- | :--- |
| **在你写的源码里** | 一个名字，比如 `loop` |
| **在汇编器的眼里** | 一个**地址**（它计算出来的数字） |
| **在CPU眼里** | **不存在**。CPU只看到地址数字，根本不知道什么`loop` |

**汇编器的工作**：
1. 扫描你的代码，看到 `loop:`，记下当前位置的地址（比如 `0x1000`）
2. 创建一个符号表：`loop` = `0x1000`
3. 当看到 `bnez x9, loop` 时，查到 `loop` 是 `0x1000`
4. 计算出偏移量（`0x1000 - 当前指令地址`），填入机器码

**最终送给CPU的机器码里，没有任何`loop`字样，只有一个数字（跳转偏移量）。**

## 三、标记的三种主要用途

### 用途1：循环（最常用）

```assembly
    li   t0, 10          # 循环10次
    li   t1, 0           # 累加器
sum_loop:                # 👈 标记：循环开始
    add  t1, t1, t0
    addi t0, t0, -1
    bnez t0, sum_loop    # 👈 跳回标记
```

### 用途2：条件跳转（if-else）

```assembly
    beq  a0, a1, equal   # 👈 如果相等，跳转到 equal 标记
    li   a0, 0           # 不相等时执行这里
    j    end             # 👈 跳转到 end 标记
equal:                   # 👈 标记：相等时来的地方
    li   a0, 1
end:                     # 👈 标记：结束位置
```

### 用途3：跳转表 / 函数入口

```assembly
    j    main            # 👈 跳转到主函数

# 子函数
print_hello:             # 👈 标记：函数入口
    li   a0, 'H'
    # ... 函数体
    ret

# 主函数
main:                    # 👈 标记：程序入口
    call print_hello     # 👈 跳转到函数标记
```

## 四、标记的范围：局部 vs 全局

| 类型 | 写法 | 作用范围 |
| :--- | :--- | :--- |
| **全局标记** | `loop:` | 整个文件都能用，汇编器会导出 |
| **局部标记** | `.Lloop:` 或 `1:` | 只在当前文件/函数内有效，不会冲突 |

**局部标记的例子**：
```assembly
func:
    li   t0, 10
1:                       # 👈 局部标记（纯数字）
    addi t0, t0, -1
    bnez t0, 1b          # 👈 1b 表示"向后找最近的 1 号标记"
    ret
```

**数字标记的后缀**：
- `1b`：向后（backward）找最近的一个 `1:` 标记
- `1f`：向前（forward）找最近的一个 `1:` 标记

## 五、标记 vs 变量（帮你彻底分清）

| 对比项 | 标记（Label） | 变量（Variable） |
| :--- | :--- | :--- |
| **指向什么** | 指令的地址 | 数据的地址 |
| **放在哪里** | 代码段（.text） | 数据段（.data） |
| **能写吗？** | 不能。不能修改标记指向的地址 | 能。变量的值可以改变 |
| **CPU能看到吗？** | 看不到。CPU只看到地址数字 | 能看到。CPU会读写这个地址 |
| **例子** | `loop: addi x9,x9,-1` | `x: .word 10` |

```assembly
.data
count: .word 10          # 👈 这是一个变量，存在内存里

.text
loop:                    # 👈 这是一个标记，标在指令前面
    lw   t0, count       # 读取变量 count 的值
    addi t0, t0, -1
    sw   t0, count       # 写回变量
    bnez t0, loop        # 跳回标记
```

## 六、一个完整的例子：计算 1+2+...+100

```assembly
.text
.globl main              # 告诉汇编器 main 是全局的，供链接器使用

main:
    li   t0, 1           # i = 1
    li   t1, 100         # 上限
    li   t2, 0           # sum = 0

sum_loop:                # 👈 标记：循环开始
    add  t2, t2, t0      # sum += i
    addi t0, t0, 1       # i++
    ble  t0, t1, sum_loop # 👈 如果 i <= 100，跳回标记

    mv   a0, t2          # 返回值存入 a0
    li   a7, 93          # exit 系统调用号
    ecall                # 退出程序
```

**标记 `sum_loop` 代表的地址**：就是 `add t2, t2, t0` 这条指令在内存中的地址。

## 七、常见错误与技巧

| 错误 | 说明 | 正确写法 |
| :--- | :--- | :--- |
| 忘记冒号 | `loop` | `loop:` |
| 标记重名 | 两个地方都叫 `loop:` | 用不同名字，或用局部标记 |
| 跳转到不存在的标记 | `bnez t0, endd`（拼错了） | 检查拼写 |
| 标记后没有指令 | `loop:`（下一行是空） | 标记后面必须跟着指令或数据 |

**技巧**：
- 循环标记常用 `1:` `2:` 这种数字标记，避免起名烦恼
- 函数入口用有意义的英文名：`main:` `print_num:` `handle_irq:`
- 跳转表用 `jump_table:` `.word case0, case1, case2`

## 总结表

| 你的问题 | 答案 |
| :--- | :--- |
| **标记是什么？** | 给某个内存地址起的名字 |
| **标记会变成机器码吗？** | 不会。标记只在汇编阶段存在 |
| **CPU能看到标记吗？** | 看不到。CPU只看到地址数字 |
| **怎么定义一个标记？** | 名字后面加冒号，如 `loop:` |
| **怎么使用标记？** | 在跳转指令中写标记名，如 `beq x1, x2, loop` |
| **标记和变量一样吗？** | 不一样。标记标在指令前，变量存数据 |
| **两个标记能重名吗？** | 全局不能，局部可以（用数字标记）|

**一句话记忆**：标记是你在地上钉的**钉子**，给钉子起个名字（`loop`），然后用 `beq` 这种“橡皮筋”套回去。CPU看不见钉子，只记得要跳回那个位置。

你之前问的 `loop` 就是这个东西。你现在已经理解了它的本质！
