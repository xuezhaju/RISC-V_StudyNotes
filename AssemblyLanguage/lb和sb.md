## `lb` 和 `sb`：字节版的加载/存储指令

`lb` 和 `sb` 是 `lw`/`sw` 的**字节版本**，用于处理内存中的**单个字节**（8位）而不是一个字（32位）。

---

## 一、基本语法

```assembly
lb rd, offset(rs1)   # Load Byte：从内存加载1个字节（符号扩展至32位）
lbu rd, offset(rs1)  # Load Byte Unsigned：加载1个字节（零扩展）
sb rs2, offset(rs1)  # Store Byte：将1个字节存入内存
```

| 指令 | 操作 | 扩展方式 |
| :--- | :--- | :--- |
| `lb` | 加载1字节 | **符号扩展**（保留正负号） |
| `lbu` | 加载1字节 | **零扩展**（高位补0） |
| `sb` | 存储1字节 | 只写入低8位 |

---

## 二、为什么需要字节操作？

处理**字符串、字节流、二进制文件**时，数据单位是字节，不是4字节的字。

```c
// C语言中的常见操作
char c = 'A';           // 一个字符占1字节
char str[] = "Hello";   // 字符串就是字节数组
unsigned char buf[10];  // 缓冲区操作
```

---

## 三、具体例子

### `sb` 示例：存储一个字符

```assembly
# 假设 x10 = 'A' (ASCII 65)
# x15 = 0x1000（内存地址）

sb x10, 0(x15)   # 把'A'存入地址0x1000
```

内存变化（小端，地址从低到高）：

| 地址 | 之前 | 之后 | 说明 |
| :--- | :--- | :--- | :--- |
| 0x1000 | 0x?? | **0x41** | 存入'A'（65的十六进制） |
| 0x1001 | 0x?? | 0x?? | 未改变 |
| 0x1002 | 0x?? | 0x?? | 未改变 |
| 0x1003 | 0x?? | 0x?? | 未改变 |

> `sb` 只改变目标地址的**1个字节**，相邻3个字节不受影响。

### `lb` 示例：加载一个字符

```assembly
lb x11, 0(x15)   # 从地址0x1000读取1字节到x11（符号扩展）

# 现在 x11 = 0x00000041（65）
```

## 四、`lb` vs `lbu`：符号扩展 vs 零扩展

这是初学者最容易犯错的地方。

### 场景：内存中存放的是 `0xFF`（即-1的补码）

```assembly
# 内存地址 0x1000 处存放着 0xFF

lb  x10, 0(x15)   # 符号扩展：x10 = 0xFFFFFFFF（-1）
lbu x11, 0(x15)   # 零扩展：  x11 = 0x000000FF（255）
```

| 指令 | 内存值 | 加载后寄存器的值 |
| :--- | :--- | :--- |
| `lb` | `0xFF` | `0xFFFFFFFF`（符号扩展到32位） |
| `lbu` | `0xFF` | `0x000000FF`（高位补0） |

**什么时候用哪个？**

| 使用场景 | 推荐指令 | 原因 |
| :--- | :--- | :--- |
| 处理**有符号**char（`signed char`） | `lb` | 保留负数值 |
| 处理**无符号**char（`unsigned char`） | `lbu` | 正确得到0-255 |
| 处理字符串（ASCII字符） | `lbu` | 字符是正数，符号扩展无关紧要，但习惯用lbu |
| 处理二进制数据 | `lbu` | 避免符号扩展污染数据 |

## 五、完整例子：实现 `strlen`

```assembly
.section .text
.globl my_strlen
# 功能：计算字符串长度（类似C的strlen）
# 参数：a0 = 字符串地址
# 返回：a0 = 长度

my_strlen:
    li a1, 0           # 长度计数器 = 0
loop:
    lbu a2, 0(a0)      # 读取1字节（用lbu，因为是字符，无符号扩展）
    beq a2, zero, done # 遇到'\0'（0）结束
    addi a0, a0, 1     # 指针++
    addi a1, a1, 1     # 长度++
    j loop
done:
    mv a0, a1          # 返回值 = 长度
    ret
```

## 六、`lw`/`sw` vs `lb`/`sb` 对比

| 特性 | `lw` / `sw` | `lb` / `sb` |
| :--- | :--- | :--- |
| 数据大小 | 4字节（字） | 1字节 |
| 地址对齐要求 | 必须是4的倍数 | **无要求**（任意地址） |
| 常见用途 | int、指针、32位数据 | char、字符串、字节流 |
| 对寄存器的影响 | 全部32位 | `lb`需扩展，`sb`只写低8位 |

## 七、内核开发中的应用

在你写内核时，字节操作无处不在：

```assembly
# 1. 串口输出一个字符（内核中最常见的操作）
.globl uart_putchar
uart_putchar:
    # a0 = 要输出的字符（低8位有效）
    lui a1, 0x1000      # 假设UART寄存器在0x10000000
    sb a0, 0(a1)        # 写1字节到UART数据寄存器
    ret

# 2. 内存设置（memset的汇编实现）
# 把指定内存区域全部设置为某个字节值
# a0 = 地址, a1 = 值, a2 = 长度
memset:
    add t0, a0, a2      # 结束地址
loop:
    beq a0, t0, done
    sb a1, 0(a0)        # 存1字节
    addi a0, a0, 1
    j loop
done:
    ret
```

## 八、动手验证

```bash
cat > byte_test.s << 'EOF'
.section .text
.globl _start
_start:
    # 准备数据
    li x10, 0x41424344   # "DCBA"（小端：44 43 42 41）
    la x11, buffer       # 内存地址
    
    sw x10, 0(x11)       # 存字：地址0-3变成44 43 42 41
    li x12, 'X'          # 88
    sb x12, 1(x11)       # 改第二个字节：地址1变成58
    
    # 现在内存：44 58 42 41  → 字符串 "B?XD" 实际是 0x44,0x58,0x42,0x41
    # 退出
    li a7, 93
    li a0, 0
    ecall

.section .data
buffer:
    .space 4
EOF

riscv64-elf-gcc -nostdlib -o byte_test byte_test.s
qemu-riscv64 ./byte_test   # 没有输出，但不会报错
```

---

**一句话总结**：`lb`/`lbu`/`sb` 是 `lw`/`sw` 的字节操作版本，处理字符和字节流时必用。`lbu`（零扩展）处理无符号数据，`lb`（符号扩展）处理有符号数据。
